# Merchant Adapters

BlindNav currently handles merchant interaction through the backend plus browser-runtime layers rather than through a standalone adapter package.

Current reality:

- primary demo target is `amazon.in`
- backup contingencies are bounded and not presented as equally mature merchant adapters
- merchant behavior is grounded in runtime interaction, observation, and verification rather than static hardcoded flows

This package directory is reserved if merchant-specific contracts or extraction boundaries become worth separating later.
